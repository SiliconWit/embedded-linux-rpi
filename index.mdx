Microcontrollers run one program. Embedded Linux runs an entire operating system on a chip smaller than a coin. The Raspberry Pi Zero 2 W costs about 15 USD and runs a full Linux kernel with Wi-Fi, USB, and enough processing power for computer vision, web servers, and database logging. This course teaches you to build, customize, and ship Linux-based embedded systems from the ground up. #EmbeddedLinux #RaspberryPi #LinuxKernel

## Why Embedded Linux When We Already Have Microcontrollers?

The previous courses in this series (ATmega328P, STM32, ESP32, RPi Pico, RTOS) teach you to read sensors, drive actuators, and communicate over UART/SPI/I2C on bare metal. That is powerful, but there are entire categories of embedded products that microcontrollers simply cannot handle:

<CardGrid>
  <Card title="What Microcontrollers Cannot Do" icon="warning">
    Run a database to store months of sensor history. Serve a full web dashboard with authentication. Process camera images with OpenCV. Run Python or Node.js scripts. Connect to USB peripherals with standard drivers. Host an SSH server for remote maintenance. Support over-the-air updates with rollback. These all require an operating system.
  </Card>

  <Card title="What This Course Builds (That MCU Courses Cannot)" icon="star">
    A systemd-managed sensor daemon that auto-starts, recovers from crashes, logs structured data to journald, and communicates over Unix sockets. A custom kernel stripped to boot in under 4 seconds. A production-grade Yocto image with A/B OTA updates, license compliance, and an SDK for your team. None of this is possible on a microcontroller.
  </Card>

  <Card title="The Industrial Reality" icon="approve-check">
    Industrial gateways, medical devices, smart cameras, point-of-sale terminals, and automotive infotainment all run embedded Linux. Companies hiring for these roles expect you to know kernel configuration, device trees, Buildroot, and Yocto. Bare-metal skills are the foundation; embedded Linux is where the higher-paying jobs are.
  </Card>

  <Card title="Not Raspbian with apt-get" icon="setting">
    This is not a "use Raspberry Pi OS" course. You will cross-compile your own kernel, write your own device tree overlays, develop a real kernel module (character device driver), build a minimal image with Buildroot, and produce a shippable Yocto image. You control every byte on the SD card.
  </Card>
</CardGrid>

## Course Structure

Each lesson follows a consistent cycle:

<Steps>
1. **The System Problem**
   A real embedded Linux challenge. Not "install Raspbian" but "build a minimal Linux that boots to your application in 4 seconds."

2. **Linux Internals**
   How the relevant subsystem works: boot sequence, device model, virtual filesystem, kernel build system.

3. **Build and Configure**
   Cross-compile on your PC, configure with menuconfig or devicetree, and deploy to the Pi Zero 2 W.

4. **Deploy and Test**
   Flash the SD card, boot, connect via serial console, and verify the system behaves as designed.

5. **Production Considerations**
   What changes when this is not a hobby project: read-only root, watchdog, logging, update mechanisms.
</Steps>

## Getting Started

<Steps>
1. **Get a Raspberry Pi Zero 2 W**, a microSD card, and a **USB-serial adapter** (CP2102 or FTDI). Total cost: about $22.

2. **Use a Linux PC for development.** Cross-compilation and kernel building work best on a native Linux machine or VM. Ubuntu 22.04+ recommended. Lessons 6 and 8 (Buildroot, Yocto) require significant disk space (30GB+).

3. **Connect via serial console**, not HDMI. The serial adapter gives you boot messages, kernel logs, and a root shell. This is how embedded Linux development actually works.

4. **Start with Lesson 1.** Cross-compile a binary on your PC and run it on the Pi. This immediately establishes the embedded Linux development workflow.

5. **Work through lessons in order.** Each lesson builds on the system configuration from previous lessons.
</Steps>

:::tip[Desktop Linux is not embedded Linux]
If you have used Raspbian/Raspberry Pi OS, that is desktop Linux on a small board. Embedded Linux means you control everything: which kernel modules load, which services start, what is in the root filesystem, and how the system boots. This course teaches that difference from Lesson 1.
:::
